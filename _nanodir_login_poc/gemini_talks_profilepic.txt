how do I set profile picture of gnome desktop account in commandline?
Gemini의 응답
Changing your GNOME profile picture via the command line is a bit more involved than clicking a button in Settings, as GNOME doesn't have a single "magic" command for this. Instead, it relies on the AccountsService daemon.

Here is the most reliable way to do it using dbus-send.

The Method: Using DBus
GNOME reads user profile data through the org.freedesktop.Accounts interface. You can manually tell this service to update your icon file path.

1. Prepare your image

Ensure your image is in a standard format (PNG or JPG) and located in a place the system can access. A good practice is to put it in your home folder or a dedicated icons folder.

2. Run the command

Replace /path/to/your/image.png with the absolute path to your photo.

Bash
dbus-send --system --dest=org.freedesktop.Accounts --print-reply \
/org/freedesktop/Accounts/User$(id -u) \
org.freedesktop.Accounts.User.SetIconFile string:"/path/to/your/image.png"
Why this works:

--system: Connects to the system bus where user account management lives.

$(id -u): Automatically inserts your current User ID (usually 1000), ensuring you target the correct account.

SetIconFile: This is the specific method that updates the pointer to your profile picture.

Alternative: The "Manual" File Method
If the DBus command feels like overkill, you can also place the image directly where the service looks for it.

Rename your desired image to .face (no extension, just the dot and the word "face").

Move it to the root of your home directory: ~/.face.

Restart your session (log out and back in).

Note: Modern versions of GNOME prioritize the AccountsService path (the DBus method above), but the ~/.face method is a classic fallback that often still works on many distributions.

Verifying the Change
To check if the system has registered the new path correctly, you can query the service:

Bash
dbus-send --system --dest=org.freedesktop.Accounts --print-reply --type=method_call \
/org/freedesktop/Accounts/User$(id -u) \
org.freedesktop.DBus.Properties.Get string:"org.freedesktop.Accounts.User" string:"IconFile"

Gemini의 응답
The "bad formed" or "invalid format" error when running this as root is a very common trap! It almost always happens because of one of two reasons:

Permission/Pathing Issues: If the image is sitting in /root or a private user directory (like a chmod 700 home folder), the AccountsService daemon cannot read it. When it fails to read the file, it misleadingly throws a "bad formed" or "not a valid image" error.

The UID Trap: If you ran $(id -u) while acting as root (via sudo or a root shell), it resolved to 0 (the root user). You were inadvertently trying to set the profile picture for root, not your actual desktop user.

Here is a robust Bash script that fixes both of these issues. It asks for the username you want to target, copies the image to a temporary, fully readable location to bypass the permission error, applies the change, and cleans up.

The Automation Script
Save the following code to a file, for example, set-profile-pic.sh.

Bash
#!/bin/bash

# 1. Ensure the script is running as root
if [ "$EUID" -ne 0 ]; then
  echo "Error: Please run this script as root (e.g., using sudo)."
  exit 1
fi

# 2. Check for correct number of arguments
if [ "$#" -ne 2 ]; then
  echo "Usage: $0 <target_username> <path_to_image>"
  echo "Example: $0 john /home/john/Downloads/avatar.jpg"
  exit 1
fi

TARGET_USER=$1
IMAGE_PATH=$2

# 3. Verify the target user exists and grab their exact UID
if ! id "$TARGET_USER" >/dev/null 2>&1; then
  echo "Error: User '$TARGET_USER' does not exist."
  exit 1
fi
TARGET_UID=$(id -u "$TARGET_USER")

# 4. Verify the image file exists
if [ ! -f "$IMAGE_PATH" ]; then
  echo "Error: Image file '$IMAGE_PATH' not found."
  exit 1
fi

# 5. The "Bad Formed" Fix:
# Copy the file to /tmp and make it world-readable so AccountsService
# doesn't get blocked by strict directory permissions (like /root/ or ~/.ssh/)
TMP_IMAGE="/tmp/gnome_avatar_${TARGET_USER}_$(basename "$IMAGE_PATH")"
cp "$IMAGE_PATH" "$TMP_IMAGE"
chmod 644 "$TMP_IMAGE"

echo "Setting profile picture for $TARGET_USER (UID: $TARGET_UID)..."

# 6. Execute the DBus command targeting the specific user's UID
dbus-send --system --dest=org.freedesktop.Accounts --print-reply \
  "/org/freedesktop/Accounts/User${TARGET_UID}" \
  org.freedesktop.Accounts.User.SetIconFile string:"$TMP_IMAGE"

DBUS_STATUS=$?

# 7. Evaluate success and clean up the temporary file
if [ $DBUS_STATUS -eq 0 ]; then
  echo "Success! Profile picture updated."
else
  echo "Failed to set profile picture. DBus returned an error."
  echo "If it still says 'bad formed', your JPG might actually be a WebP file with a .jpg extension."
fi

rm -f "$TMP_IMAGE"
How to use it
Make the script executable:

Bash
chmod +x set-profile-pic.sh
Run it with sudo, providing the username and the image path:

Bash
sudo ./set-profile-pic.sh yourusername /path/to/the/picture.jpg

Here is the Python equivalent of the bash script.

I've used standard Python libraries (os, pwd, shutil, subprocess) so you don't need to install any extra packages (like dbus-python, which can sometimes be a hassle to set up in a root environment). It handles the arguments, user lookup, file permission workaround, and the DBus call natively.

The Python Script
Save this code to a file, for example, set_profile_pic.py.

Python
#!/usr/bin/env python3

import os
import sys
import pwd
import shutil
import subprocess
import argparse

def main():
    # 1. Ensure the script is running as root
    if os.geteuid() != 0:
        print("Error: Please run this script as root (e.g., using sudo).", file=sys.stderr)
        sys.exit(1)

    # 2. Parse command-line arguments
    parser = argparse.ArgumentParser(description="Set GNOME profile picture for a specific user.")
    parser.add_argument("username", help="The target username")
    parser.add_argument("image_path", help="The absolute path to the image file")
    args = parser.parse_args()

    # 3. Verify the target user exists and grab their exact UID
    try:
        user_info = pwd.getpwnam(args.username)
        target_uid = user_info.pw_uid
    except KeyError:
        print(f"Error: User '{args.username}' does not exist.", file=sys.stderr)
        sys.exit(1)

    # 4. Verify the image file exists
    if not os.path.isfile(args.image_path):
        print(f"Error: Image file '{args.image_path}' not found.", file=sys.stderr)
        sys.exit(1)

    # 5. The "Bad Formed" Fix:
    # Copy the file to /tmp and make it world-readable so AccountsService
    # doesn't get blocked by strict directory permissions.
    file_name = os.path.basename(args.image_path)
    tmp_image = f"/tmp/gnome_avatar_{args.username}_{file_name}"

    try:
        shutil.copy(args.image_path, tmp_image)
        os.chmod(tmp_image, 0o644)  # rw-r--r--
    except Exception as e:
        print(f"Error preparing image file: {e}", file=sys.stderr)
        sys.exit(1)

    print(f"Setting profile picture for {args.username} (UID: {target_uid})...")

    # 6. Execute the DBus command targeting the specific user's UID
    dbus_cmd = [
        "dbus-send",
        "--system",
        "--dest=org.freedesktop.Accounts",
        "--print-reply",
        f"/org/freedesktop/Accounts/User{target_uid}",
        "org.freedesktop.Accounts.User.SetIconFile",
        f"string:{tmp_image}"
    ]

    result = subprocess.run(dbus_cmd, capture_output=True, text=True)

    # 7. Evaluate success
    if result.returncode == 0:
        print("Success! Profile picture updated.")
    else:
        print("Failed to set profile picture. DBus returned an error:", file=sys.stderr)
        print(result.stderr.strip() or result.stdout.strip(), file=sys.stderr)
        print("\nNote: If the error says 'bad formed', your image file might have a fake extension (e.g., a WebP file named .jpg).", file=sys.stderr)

    # 8. Clean up the temporary file
    try:
        os.remove(tmp_image)
    except OSError:
        pass

if __name__ == "__main__":
    main()

Bash

Run it with sudo, providing the username and the image path:

Bash

Gemini의 응답
Ah, this is a classic DBus quirk!

That error means AccountsService is being lazy. If User 1002 was recently created, or if they haven't logged into the graphical desktop recently, AccountsService isn't actively tracking them in memory. Because it isn't tracking them, the D-Bus path /org/freedesktop/Accounts/User1002 literally doesn't exist yet.

To fix this, we have to "wake up" AccountsService and force it to load the user's profile into D-Bus before we try to set the picture. We do this by calling the FindUserByName method first.

Here is the updated Python script with that crucial "wake up" step added.

    dbus_cmd = [
        "dbus-send",
        "--system",
        "--dest=org.freedesktop.Accounts",
        "--print-reply",
        f"/org/freedesktop/Accounts/User{target_uid}",
        "org.freedesktop.Accounts.User.SetIconFile",
        f"string:{tmp_image}"

        print(result.stderr.strip() or result.stdout.strip(), file=sys.stderr)

    # 9. Clean up the temporary file
    try:
        os.remove(tmp_image)
    except OSError:
        pass

if __name__ == "__main__":
    main()

I added Step 6. It pings the main /org/freedesktop/Accounts interface and asks it to FindUserByName. This forces the daemon to generate the /org/freedesktop/Accounts/User1002 path so that Step 7 has something to actually attach the image to.

Error org.freedesktop.Accounts.Error.Failed: file '/tmp/profile.png' is not a regular file

Now it says this
Gemini의 응답
This is actually a brilliant security feature of modern Linux working against us! You have just run into systemd's PrivateTmp.

Here is what is happening: The accounts-daemon (the service running AccountsService) runs with strict sandboxing. One of those sandboxing rules is PrivateTmp=yes. This means the daemon gets its own, completely isolated, invisible /tmp directory.

When your script puts the image in /tmp/profile.png, the root user can see it perfectly fine. But when AccountsService looks inside its /tmp directory, the file isn't there. So it throws a confusing "not a regular file" or "no such file" error.

To fix this, we just need to stage the file in a directory that is not hidden by systemd sandboxing. The best place is actually /var/lib/AccountsService/icons/, which is where the daemon natively expects to find and store these images anyway.

The Fix: Bypassing PrivateTmp
Here is the corrected Python script. I've updated the staging directory from /tmp to /var/lib/AccountsService/icons/.

Python
#!/usr/bin/env python3

import os
import sys
import pwd
import shutil
import subprocess
import argparse

def main():
    # 1. Ensure the script is running as root
    if os.geteuid() != 0:
        print("Error: Please run this script as root (e.g., using sudo).", file=sys.stderr)
        sys.exit(1)

    # 2. Parse command-line arguments
    parser = argparse.ArgumentParser(description="Set GNOME profile picture for a specific user.")
    parser.add_argument("username", help="The target username")
    parser.add_argument("image_path", help="The absolute path to the image file")
    args = parser.parse_args()

    # 3. Verify the target user exists and grab their exact UID
    try:
        user_info = pwd.getpwnam(args.username)
        target_uid = user_info.pw_uid
    except KeyError:
        print(f"Error: User '{args.username}' does not exist.", file=sys.stderr)
        sys.exit(1)

    # 4. Verify the image file exists
    if not os.path.isfile(args.image_path):
        print(f"Error: Image file '{args.image_path}' not found.", file=sys.stderr)
        sys.exit(1)

    # 5. The "PrivateTmp" Fix:
    # Use the native AccountsService icons directory instead of /tmp to bypass systemd sandboxing.
    icons_dir = "/var/lib/AccountsService/icons"
    os.makedirs(icons_dir, exist_ok=True)

    file_name = os.path.basename(args.image_path)
    stage_image = os.path.join(icons_dir, f"staged_{args.username}_{file_name}")

    try:
        shutil.copy(args.image_path, stage_image)
        os.chmod(stage_image, 0o644)  # rw-r--r--
    except Exception as e:
        print(f"Error preparing image file: {e}", file=sys.stderr)
        sys.exit(1)

    print(f"Waking up AccountsService for {args.username}...")

    # 6. Force AccountsService to load the user object into DBus
    wake_cmd = [
        "dbus-send",
        "--system",
        "--dest=org.freedesktop.Accounts",
        "--print-reply",
        "/org/freedesktop/Accounts",
        "org.freedesktop.Accounts.FindUserByName",
        f"string:{args.username}"
    ]
    subprocess.run(wake_cmd, capture_output=True)

    print(f"Setting profile picture for {args.username} (UID: {target_uid})...")

    # 7. Execute the DBus command targeting the specific user's UID
    dbus_cmd = [
        "dbus-send",
        "--system",
        "--dest=org.freedesktop.Accounts",
        "--print-reply",
        f"/org/freedesktop/Accounts/User{target_uid}",
        "org.freedesktop.Accounts.User.SetIconFile",
        f"string:{stage_image}"
    ]

    result = subprocess.run(dbus_cmd, capture_output=True, text=True)

    # 8. Evaluate success
    if result.returncode == 0:
        print("Success! Profile picture updated.")
    else:

        print(result.stderr.strip() or result.stdout.strip(), file=sys.stderr)

    # 9. Clean up the staged file
    # (AccountsService usually copies it internally, so we don't need to leave our staged copy)
    try:
        os.remove(stage_image)
    except OSError:
        pass

if __name__ == "__main__":
    main()
